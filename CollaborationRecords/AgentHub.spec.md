# AgentHub Spec

- 版本：v1
- 日期：2026-05-21
- 范围：`AgentHub` 新项目首发方案，聚焦协作工作台、协作型任务闭环、文档和 PPT 产物。

## 1. 项目定义

AgentHub 是一个面向复杂内容任务的多 Agent 协作工作台。

它不是传统聊天产品，而是一个把用户需求转化为可拆解任务、并行协作结果和可编辑产物的生产界面。

## 2. 首发目标

首发版本只做一条完整闭环：

1. 用户输入协作型任务
2. 主 Agent 进行理解和拆解
3. 子 Agent 并行执行
4. 系统汇总文档和 PPT 产物
5. 用户在同一工作台内预览、修改、追问、再次协作

## 3. 产品形态

### 3.1 核心形态

- 协作工作台
- IM 式入口
- 任务流可视化
- 产物预览与编辑并存

### 3.2 首发产物

- 文档
- PPT

### 3.3 产物能力优先级

1. 能生成
2. 能预览
3. 能局部修改
4. 能重新生成局部内容
5. 能保存版本

## 4. 角色定义

### 4.1 用户

用户提出复杂内容需求，查看协作过程，修改产物并继续驱动下一轮协作。

### 4.2 主 Agent

主 Agent 是协调器，负责：

- 理解任务目标
- 判断任务复杂度
- 拆分子任务
- 调度子 Agent
- 聚合结果
- 处理失败与冲突

### 4.3 子 Agent

子 Agent 负责职责单一的子任务。

建议的首发分工：

- 文档 Agent：章节结构、正文组织、总结提炼
- PPT Agent：页面结构、页标题、要点拆分、演示表达
- 补充 Agent：风险点、亮点、细节优化、措辞润色

## 5. MVP 功能

### 5.1 必做

- 会话入口
- 自然语言任务输入
- 主 Agent 任务规划
- 子 Agent 并行协作
- 协作状态可视化
- 文档生成与预览
- PPT 生成与预览
- 局部修改与二次追问
- 协作记录沉淀

### 5.2 可延后

- 完整代码编辑器
- 网页产物深度预览
- 更复杂的版本对比
- 多端同步
- 更完整的 Agent 市场
- 用户自定义 Agent 的高级配置
- 团队权限管理

### 5.3 暂不做

- 通用型全量创作平台
- 所有产物类型一开始全覆盖
- 复杂的多人实时协同编辑
- 重配置流程系统
- 非核心装饰性功能

## 6. 页面结构

### 6.1 左侧

- 会话列表
- 当前任务列表
- Agent 列表
- 能力标签

### 6.2 中间

- 用户消息
- 主 Agent 规划
- 子 Agent 回传
- 任务拆解卡片
- 执行状态卡片
- 失败 / 重试 / 降级提示

### 6.3 右侧

- 文档预览
- PPT 预览
- 版本历史
- 局部修改入口
- 重新生成入口

## 7. 任务流

标准流程：

1. 用户输入协作型任务
2. 主 Agent 识别目标
3. 主 Agent 拆分子任务
4. 多个子 Agent 并行执行
5. 系统汇总结果
6. 生成文档与 PPT 产物
7. 用户修改或追问
8. 主 Agent 基于新上下文继续协作

## 8. 设计原则

- 先讲清楚协作闭环，再扩展功能
- 先做可见的任务流，再做复杂的编辑能力
- 先做文档和 PPT，再考虑其他产物
- 先保住工作台感，再考虑多端
- 先保住协作质量，再谈更大范围接入
