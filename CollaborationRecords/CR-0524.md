## 1. 当前首发主线

AgentHub 的首发不以“聊天产品”作为主叙事，而以“协作工作台”作为核心产品形态。

主线是：

`首页任务列表 -> 进入任务工作台 -> 多 Agent 协作 -> 文档 / PPT 产物 -> 二级入口查看 Agent 记录与 rules -> 针对单个 Agent 调整协作方式`

这条主线对应当前已经确定的几个关键判断：

- 首页默认打开的是全新的任务列表
- 每个任务有独立产物区
- 同一任务内的文档和 PPT 放在同一个产物区中
- 不同任务之间的产物区彼此隔离
- 协作记录放在任务详情页内的二级入口
- 协作记录重点不是任务复盘本身，而是每个 Agent 的工作记录与 rules
- 用户可以针对单个 Agent 单独制定 rules

## 3. IA 主线说明

这版 IA 的核心不是“页面多”，而是“主线清晰”：

1. 首页只负责找任务和进任务
2. 任务详情页是整个产品的主工作台
3. 右侧产物区按任务隔离，任务内共享文档和 PPT
4. 协作记录不作为首页主模块，而是任务详情页内的二级入口
5. 协作记录的重点是每个 Agent 的工作记录和 rules
6. 用户可以直接针对某一个 Agent 修改 rules
7. 系统预留只做扩展，不抢首发主线

## 4. 当前阶段的设计判断

- 首页应该像任务中枢，而不是消息中枢
- 工作台应该像协作生产工具，而不是普通聊天工具
- 文档和 PPT 应该共享同一个任务上下文
- 协作记录应该服务于 Agent 调优，而不是只做历史回顾
- rules 应该支持单 Agent 级别的单独配置
