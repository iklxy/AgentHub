### 消息操作 1.0版本

# 1.代码复制（已实现）

# 2.引用

- 用户在某一条消息的基础上进行内容的追加，如：
Agent：这个算法的时间复杂度是o（n）
用户引用该条消息：为什么是o（n）

- 用提示词拼接的方式，把新的消息组合为，类似如下形式：
前文中提到“<cite message(在这里是“这个算法的时间复杂度是o（n）)>”，<user message（在这里是 为什么是o（n）>？
即，前文中提到这个算法的时间复杂度是o（n），为什么是o（n）？

- 实现接口：POST /api/messages/{message_id}/quote
- 每一条消息在数据库中都有id字段，在引用某一条消息时，根据被引用消息的id字段去数据库的content_md字段中取出对应的消息内容

# 3.回复

- 用户针对某一个问题进行回复，要求Agent在被回复的问题的基础上进行回答，即新消息的语义目标就是被回复的消息

用户对 SQL可以用来操作数据库 进行 reply，reply的内容是 你这里所说的SQL是什么？

- 用提示词拼接的方式
请基于以下消息<replry message>，回到以下内容<user message>

- 实现接口:POST /api/messages/{message_id}/reply
- 每一条消息在数据库都有id字段，在回复某一条消息是，根据被回复的id字段去查消息的内容，然后在新消息的reply_to_message_id字段填入被回复的消息的id
