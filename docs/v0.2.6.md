### 模式转移

## 1.当前的系统调用Claude code的方式是go通过起一个python的子进程，通过claude -p --resume的形式持续的进行会话
- 这样的交互形式有一些弊端
- 1.claude code在请求权限的时候，请求无法通过后端服务抵达前端，来供用户进行选择
- 2.后续指定claude code使用特定的工具以及开放特定的权限的时候麻烦

## 2.模式转移：用CLI直接调用，转为调用claude code SDK
参考文档网站：https://code.claude.com/docs/zh-CN/agent-sdk#python

# （1）持续会话
# 参考的代码形式如下：

session = db.get_agent_session(conversation_id) # 代表前端当前所属的session的id，当前不需要从数据库中查，只是供你做参考
options = ClaudeAgentOptions(
            cwd=session.cwd,
            resume=session.claude_session_id, # 前端React组件内存中存储的session_id
            allowed_tools=["Read", "Glob", "Grep", "Edit", "Write"], # 这是后续的权限部分，暂时先不做
        )

async for message in query(prompt=user_message, options=options): # user_message就是用户在前端输入的信息或者用程序拼接出来的信息

    final_text = message.result # 这是claude code的回答，也就是返回给用户的回答

# (2)通过SDK可以来对Claude code的权限申请以及工具调用申请作出审批（这一步先不做）
