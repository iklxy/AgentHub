# 权限批准

## 1. 文件搜索/读取类

- Read
- Glob
- Grep
- ls

以上全部默认放行（`allowed_tools` 白名单）。

## 2. 编辑类

- Edit
- MultiEdit
- Write

以上需要走回调函数，由前端用户审批。

## 3. Shell 命令类

- Bash

放行以下命令（Python 端就近判断，不通知前端）：
- git status
- git diff
- ls
- cat
- pwd
- find
- grep

需回调确认的命令（通知前端审批）：
- npm test / pnpm test / pytest / go test / cargo test 等测试命令
- curl / wget 等网络请求命令

直接拒绝的命令（Python 端就近判断）：
- Read(**/.env*)
- Read(**/.ssh/**)
- Bash(rm -rf *)
- Bash(git push *)
- Bash(sudo *)
- Bash(ssh *)
- Bash(scp *)
- Bash(curl * | bash)

## 4. 三层交互协议

### 架构

```
┌──────────┐  stdout(JSONL)   ┌──────────┐  SSE push / REST   ┌──────────┐
│  Python  │ ────────────────→│    Go    │ ─────────────────→│ Frontend │
│   SDK    │←────────────────│ Backend  │←─────────────────│  (React) │
└──────────┘  stdin(JSONL)    └──────────┘  REST response     └──────────┘
```

- Python ↔ Go：双向 JSONL 管道通信（每行一个完整 JSON，`\n` 分隔）
- Go ↔ Frontend：SSE 推送审批请求 + REST 回传用户决定

---

### 4.1 Python → Go（stdout JSONL）

#### 权限申请

```json
{
  "type": "permission_request",
  "requestId": "perm_<sessionId>_<seq>",
  "toolName": "Bash",
  "input": {
    "command": "npm test",
    "description": "Run project test suite",
    "timeout": 120000
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `type` | string | 固定值 `"permission_request"` |
| `requestId` | string | 唯一请求 ID，格式 `perm_{sessionId}_{seq}` |
| `toolName` | string | 工具名：`Bash` `Write` `Edit` `MultiEdit` |
| `input` | object | 工具原始参数，格式随 toolName 变化（见下方） |

#### 各工具 input 格式

**Bash**：
```json
{
  "command": "npm install",
  "description": "Install project dependencies",
  "timeout": 120000
}
```

**Write**：
```json
{
  "file_path": "/workspace/project/src/new-file.ts",
  "content": "export const hello = 'world'\n"
}
```

**Edit**：
```json
{
  "file_path": "/workspace/project/src/main.ts",
  "old_string": "return result",
  "new_string": "return result.toUpperCase()"
}
```

#### 最终结果

```json
{
  "type": "agent_result",
  "result": "已完成代码重构，修改了 3 个文件...",
  "session_id": "sdk-abc123"
}
```

### 4.2 Go → Python（stdin JSONL）

#### 允许

```json
{
  "type": "permission_response",
  "requestId": "perm_<sessionId>_<seq>",
  "behavior": "allow",
  "updatedInput": {
    "command": "npm test -- --grep=login",
    "description": "Run project test suite",
    "timeout": 120000
  }
}
```

#### 拒绝

```json
{
  "type": "permission_response",
  "requestId": "perm_<sessionId>_<seq>",
  "behavior": "deny",
  "message": "当前不允许执行测试命令"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `type` | string | 固定值 `"permission_response"` |
| `requestId` | string | 对应权限申请的 requestId |
| `behavior` | string | `"allow"` 或 `"deny"` |
| `updatedInput` | object | allow 时必填，允许前端修改参数后回传 |
| `message` | string | deny 时必填，告知 Claude 拒绝原因 |

### 4.3 Go → Frontend（SSE 推送）

**端点**：`GET /api/sessions/{sessionId}/permissions/events`

SSE 事件格式：

```
event: permission_request
data: {
  "requestId": "perm_fd3c3ae9_1",
  "sessionId": "fd3c3ae9-a387-4936-83be-3b125ff01d9c",
  "toolName": "Bash",
  "input": {
    "command": "npm test",
    "description": "Run project test suite",
    "timeout": 120000
  },
  "createdAt": "2026-05-30T10:00:00Z"
}
```

#### 前端 UI 字段映射

| JSON 字段 | 展示位置 |
|---|---|
| `toolName` | 卡片标题："Bash 命令审批" / "文件写入审批" / "文件编辑审批" |
| `input.command` (Bash) | 代码块展示完整命令（带语法高亮） |
| `input.description` (Bash) | 副标题或说明行 |
| `input.file_path` (Write/Edit) | 文件路径 breadcrumb |
| `input.content` (Write) | 代码 diff 预览 |
| `input.old_string` → `input.new_string` (Edit) | 左右对比 diff |

已有的前端审批卡片组件：`frontend/src/components/workspace/runtime-approval-card.tsx`

### 4.4 Frontend → Go（REST）

**端点**：`POST /api/sessions/{sessionId}/permissions/{requestId}/respond`

请求体（允许）：
```json
{
  "behavior": "allow",
  "updatedInput": {
    "command": "npm test -- --grep=login",
    "description": "Run project test suite",
    "timeout": 120000
  }
}
```

请求体（拒绝）：
```json
{
  "behavior": "deny",
  "message": "不要执行测试"
}
```

### 4.5 权限判断流程

```
can_use_tool 回调入口
├─ toolName in [Read, Glob, Grep]?
│   → 直接 PermissionResultAllow（不通知前端）
│
├─ toolName == "Bash"?
│   ├─ command 命中 AUTO_ALLOW_COMMANDS
│   │   → 直接 Allow
│   ├─ command 命中 AUTO_DENY_COMMANDS
│   │   → 直接 Deny
│   └─ 其他
│       → 走审批管道（stdout→Go→SSE→前端→REST→Go→stdin）
│
├─ toolName in [Edit, Write, MultiEdit]?
│   → 走审批管道
│
└─ file_path 匹配 .env / .ssh 等敏感路径?
    → 直接 Deny（最高优先级）
```

### 4.6 完整时序

```
Python                   Go                        Frontend
  │                       │                           │
  │── permission_request──→│                           │
  │   (stdout JSONL)      │── SSE push ──────────────→│
  │                       │   显示 RuntimeApprovalCard  │
  │   (进程保持运行)       │                           │
  │                       │                           │← 用户选择 allow/deny
  │                       │                           │   （可修改命令参数）
  │                       │←── POST /respond ─────────│
  │                       │                           │
  │← permission_response ─│                           │
  │   (stdin JSONL)       │                           │
  │                       │                           │
  │ SDK can_use_tool 返回  │                           │
  │ SDK 继续执行           │                           │
  │                       │                           │
  │── permission_request──→│  （可能多轮审批）          │
  │       ...             │        ...                │
  │                       │                           │
  │── agent_result ──────→│                           │
  │                       │── 正常 HTTP 响应 ─────────→│
```

### 4.7 关键设计点

1. **双向管道**：Go 不再使用 `cmd.Output()`，改用 `cmd.StdinPipe()` + `cmd.StdoutPipe()` + `bufio.Scanner` 逐行读写 JSONL
2. **goroutine 等待**：Go 收到 `permission_request` 后，创建 channel，推送 SSE，在 goroutine 中阻塞等待 REST 回写 channel
3. **多轮支持**：一次 Agent 执行可能触发多次权限申请，管道保持打开直到 `agent_result`
4. **超时保护**：前端 5 分钟内未响应，Go 自动发送 deny 响应给 Python，结束等待
5. **就近判断**：Read/Glob/Grep 和简单 Bash 命令的放行/拒绝在 Python 端完成，减少前后端交互
